и®әж–Үйҳ…иҜ»[зІ—иҜ»]-THE CORA TENSOR COMPILER: COMPILATION FOR RAGGED TENSORS WITH MINIMAL PADDING

еҸ‘иЎЁдәҺ 2022-07-11 жӣҙж–°дәҺ 2023-09-23 йҳ…иҜ»ж¬Ўж•°пјҡ Valineпјҡ
жң¬ж–Үеӯ—ж•°пјҡ 2.1k йҳ…иҜ»ж—¶й•ҝ ≈ 2 еҲҶй’ҹ

йҷҲеӨ©еҘҮеңЁMLSys 22еҸ‘зҡ„еҸҰдёҖзҜҮж–Үз« пјҢи®Ізҡ„жҳҜеҰӮдҪ•жҸҗй«ҳиҫ“е…ҘдёҚеҜ№йҪҗж—¶зҡ„иЎЁзҺ°гҖӮ

е·Іжңүзҡ„ж–№жі•йқўеҜ№дёҚеҜ№йҪҗзҡ„ж•°жҚ®пјҢеёёеёёжҳҜpaddingеҲ°дёҖдёӘй•ҝеәҰпјҢиҝҷж ·дјҡйҖ жҲҗйўқеӨ–зҡ„и®Ўз®—гҖӮжң¬ж–Үи§ЈеҶіиҝҷдёӘй—®йўҳпјҢеҫ—еҲ°дәҶPytorchдёҠ16еҖҚзҡ„йҖҹеәҰжҸҗеҚҮпјҢTensorFlowдёҠ1.4еҖҚзҡ„жҖ§иғҪжҸҗеҚҮ

Introduction

DLжЁЎеһӢзҡ„иҫ“е…ҘеёёеёёдёҚеҜ№йҪҗпјҢеҰӮдёӢеӣҫжүҖзӨәпјҡ

жңҖиҝ‘жңүдёҖдәӣзј–иҜ‘еҷЁејҖе§Ӣж”ҜжҢҒдёҚеҜ№йҪҗзҡ„ж•°жҚ®пјҢдҪҶжҳҜиҢғеӣҙгҖҒж•ҲжһңеҫҲеҸ—йҷҗгҖӮиҖҢз»Ҹе…ёзҡ„cuDNNпјҢoneDNNеә“йғҪйҮҮз”ЁдәҶMASKпјҢPADDINGзҡ„ж–№жі•гҖӮиҝҷз§Қж–№жі•дјҡеёҰжқҘйўқеӨ–зҡ„и®Ўз®—жҚҹеӨұ(зәўиүІйғЁеҲҶ)пјҢиҖҢдё”йҡҸзқҖbatch sizeзҡ„еўһеӨ§иҝҷдёӘжҚҹеӨұдјҡжӣҙеҠ дёҘйҮҚгҖӮ

жң¬ж–ҮжҸҗеҮәдәҶдёҖеҘ—compiler-basedж–№жі•и§ЈеҶідёҚеҜ№йҪҗж•°жҚ®зҡ„й—®йўҳгҖӮзЁҖз–Ҹе’ҢеҜҶйӣҶзҡ„tensorзҡ„з ”з©¶еҫҲеӨҡпјҢдҪҶеӨ§еӨҡдёҚиғҪеә”з”ЁеҲ°rag tensor,еҗҢж—¶жң¬ж–ҮжҸҗеҮәCORA (Compiler for Ragged Tensors)пјҡ

  • Irregularity in generated codeпјҡеҸҜеҸҳзҡ„еҫӘзҺҜеёҰжқҘеҸҜеҸҳзҡ„д»Јз ҒпјҢдҪҝеҫ—GPUжҖ§иғҪдёӢйҷҚгҖӮ
    • CORAдҪҝз”Ёminimal paddingж–№жі•пјҢжқҘе№іиЎЎй«ҳж•Ҳд»Јз Ғз”ҹжҲҗе’Ңthread remapping strategies
  • Insufficient compiler mechanismsпјҡз”ұдәҺеҸҜеҸҳеҫӘзҺҜеҶ…зҡ„дҫқиө–е…ізі»еӨҚжқӮпјҢRepresenting transformationsж•ҲжһңеҫҲе·®гҖӮе№¶дё”зЁҖз–Ҹtensorзҡ„дјҳеҢ–ж–№жі•еҸҜиғҪдёҚиғҪеә”з”ЁеҲ°rag tensorпјҢеӣ дёәеҗҺиҖ…еҸҜиғҪжӣҙеҠ зЁҖз–ҸгҖӮ
    • Coraз”Ёuninterpreted functionsиЎЁзӨәеҫӘзҺҜиҫ№з•ҢпјҢжқҘи°ғеәҰoperation
  • Ill-fitting computation abstractionsпјҡжҺҘеҸЈе’ҢжҠҪиұЎдёҚеҢ№й…Қзҡ„й—®йўҳгҖӮdense compilersдёҚиғҪиЎЁиҫҫrag opпјҢsparse compilersжІЎжңүеҠһжі•з”ҹжҲҗй«ҳж•Ҳзҡ„codeгҖӮ
    • CORAз»ҷpaddingпјҢthread remappingзӯүжҸҗдҫӣдёҖеҘ—жҠҪиұЎпјҢжқҘи§ЈеҶідёҚеҢ№й…Қзҡ„й—®йўҳгҖӮ

е’Ңе·Іжңүж–№жі•еҜ№жҜ”еҰӮдёӢпјҡ

жң¬ж–Үдё»иҰҒиҙЎзҢ®пјҡ

  • жҸҗеҮәCORAпјҢз¬¬дёҖдёӘй«ҳж•Ҳи§ЈеҶіragged tensorзҡ„tensorзј–иҜ‘еҷЁ
  • дёәragged tensorжҸҗдҫӣдәҶж–°зҡ„APIпјҢжҠҪиұЎе’Ңscheduleingж–№жі•
  • иҝӣиЎҢдәҶиҜҰз»ҶиҜ„жөӢгҖӮ

CORA OVERVIEW

еҹәдәҺд»ҘдёӢзҡ„дёӨдёӘзҒөж„ҹпјҡ

  • ragged operationдёӯзҡ„дёҚеҜ№йҪҗжҖ§еңЁи®Ўз®—еүҚе°ұеҸҜд»ҘиҺ·еҸ–еҲ°
    • еҸҜд»ҘеңЁзңҹжӯЈи®Ўз®—еүҚйў„и®Ўз®—дёҚеҜ№йҪҗжҖ§еёҰжқҘзҡ„иҫ…еҠ©дҝЎжҒҜ
  • ragged tensorжҜ”иө·зЁҖз–ҸtensorпјҢеҸҜд»Ҙз”Ё$\mathcal{O}(1)$ж—¶й—ҙи®ҝй—®пјҲHASHж–№жі•иҷҪз„¶еҸҜд»Ҙ$\mathcal{O}(1)$зЁҖз–ҸtensorпјҢдҪҶдёҚеҲ©дәҺGPUжү§иЎҢпјү
    • еҸҜд»Ҙиҫ…еҠ©з”ҹжҲҗй«ҳж•Ҳзҡ„д»Јз Ғ

CORAжҖ»дҪ“жү§иЎҢеҰӮдёҠеӣҫжүҖзӨә

жңҜиҜӯ

  • variable loops or vloops пјҡеҰӮжһңеҶ…йғЁеҫӘзҺҜзҡ„иҫ№з•ҢжҳҜе’ҢеӨ–еұӮеҸҳйҮҸжңүе…ізҡ„
  • vloop nestпјҡеҢ…еҗ«еӨҡдёӘvloop
  • variable dimensionsпјҡеҪ“еӯҳеӮЁtensorдёҚз”Ёpaddingж—¶пјҢйҪҗеҪўзҠ¶еҸ–еҶідәҺеӨ–йғЁtensorгҖӮиҝҷдёӘеҪўзҠ¶з§°дёәvariable dimensionsпјҢжҲ–иҖ…vdims
  • cdimsпјҡеӣәе®ҡеҪўзҠ¶
  • ragged tensorпјҡжңүжңҖе°‘дёҖдёӘvdimзҡ„tensor

CORAвҖҷS RAGGED API

CORAзҡ„APIиҰҒжұӮеҜ№дәҺragged tensorжҸҸиҝ°еҮәvdimжҳҜжҖҺд№Ҳи®Ўз®—зҡ„пјҢз”ұеӨ–еұӮеҫӘзҺҜзҡ„д»Җд№Ҳзү№еҫҒжқҘе®ҡд№үгҖӮж»Ўи¶іжӯӨжқЎд»¶еҗҺпјҢCORAе°ұеҸҜд»ҘиҮӘеҠЁзҡ„з¬ҰеҗҲеҗҺйқўи®Ўз®—APIзҡ„е®һзҺ°гҖӮ

Scheduling Primitives

  • Loop SchedulingпјҡCORAдёҚе…Ғи®ёеҜ№vloopдәӨжҚўйЎәеәҸпјҢеӣ дёәжІЎжңүеҸ‘зҺ°жңүд»Җд№Ҳе®һйҷ…йңҖжұӮгҖӮ
  • Operation SplittingпјҡCORAеҸҜд»ҘжҠҠOPеҲҶи§ЈжҲҗеӨҡдёӘOPпјҢеҗҺйқўжҠҠжӢҶйҷӨзҡ„йғЁеҲҶе№¶иЎҢи®Ўз®—гҖӮ
  • Horizontal FusionпјҡCORAж”ҜжҢҒhfusionпјҢеҸҜд»ҘжҠҠжңҖеӨ–еұӮеҫӘзҺҜеҶ…зҡ„еҮ дёӘoperatorе№¶иЎҢи®Ўз®—
  • Loop and Storage PaddingпјҡеҜ№дәҺminimal padding(дәҢеҜ№йҪҗгҖҒеӣӣеҜ№йҪҗ)пјҢеҸҜд»ҘеҜ№йҪҗзҡ„еӯҳеӮЁпјҢе№¶дё”дёҚжөӘиҙ№еҫҲеӨҡйўқеӨ–зҡ„з©әй—ҙ
  • Tensor Dimension SchedulingпјҡCORAе…Ғи®ёз”ЁжҲ·дәӨжҚўtensorзҡ„з»ҙеәҰ
  • Load BalancingпјҡCORAе…Ғи®ёз”ЁжҲ·еҺ»иҮӘе·ұж»Ўи¶іthread remapping policyпјҢжқҘи§ЈеҶіиҝҷдёӘй—®йўҳгҖӮ
